# Contributing to Ecosystem (ELcoin Platform)

Thank you for your interest in contributing!

## Development rules
- Always create a new branch from `main`
- Describe your changes clearly in pull requests
- Run tests before sending a PR
- Never commit secrets, private keys or .env files

## Pull Request Process
1. Open an issue first (feature or bug)
2. Create a branch: feature/my-feature
3. Push your code and open a pull request
4. A reviewer must approve before merging
